# Spell Checker Roadmap

## Текущая реализация (v1.0)

### ✅ Реализовано
- Базовый spell checker на основе Levenshtein distance
- Словарь из 4100 слов (IAM Words training set)
- Оптимизация поиска по длине слова (`words_by_len`)
- Логарифмический scoring для баланса distance/frequency
- Улучшение WER: ~7% (относительное)

### Архитектура
```python
class SpellChecker:
    vocabulary: dict[str, int]           # {word: frequency}
    words_by_len: dict[int, list[str]]   # {length: [words]}
    max_edit_distance: int = 2
    min_word_length: int = 3
```

### Производительность
- Поиск: O(N/10) где N - размер словаря
- Память: ~50KB (словарь) + ~100KB (индекс по длине)
- Скорость: ~1-2ms на слово

## Roadmap

### 🔄 v2.0: Symspell Algorithm (приоритет: HIGH)

**Цель**: Ускорение поиска в 50-100x

**Подход**: Предвычисленные удаления (deletes)

**Алгоритм**:
1. При инициализации генерируем все возможные "удаления" для каждого слова
2. Храним индекс: `{delete_variant: [original_words]}`
3. При поиске генерируем удаления для входного слова
4. Получаем кандидатов из индекса (O(1))
5. Проверяем только кандидатов (10-50 вместо 4000)

**Пример**:
```python
# Словарь: ["hello", "world"]
# Предвычисленные удаления (max_distance=1):
deletes = {
    "ello": ["hello"],   # удалили 'h'
    "hllo": ["hello"],   # удалили 'e'
    "helo": ["hello"],   # удалили 'l'
    "hell": ["hello"],   # удалили 'o'
    "orld": ["world"],   # удалили 'w'
    "wrld": ["world"],   # удалили 'o'
    # ...
}

# Поиск для "helo" (опечатка в "hello"):
# 1. Генерируем удаления: ["elo", "hlo", "heo", "hel"]
# 2. Ищем в индексе: deletes["hlo"] = ["hello"]
# 3. Проверяем distance("helo", "hello") = 1 ✅
```

**Преимущества**:
- Поиск за O(1) вместо O(N)
- Проверяем только релевантных кандидатов
- Ускорение в 50-100x

**Недостатки**:
- Требует больше памяти (~5-10MB для словаря 4000 слов)
- Сложнее в реализации
- Предвычисление занимает время при инициализации (~1-2 секунды)

**Оценка трудозатрат**: 1-2 дня

**Приоритет**: HIGH (значительное ускорение)

---

### 🔄 v3.0: N-gram коррекция (приоритет: MEDIUM)

**Цель**: Контекстная коррекция слов

**Подход**: Биграммы/триграммы для учёта контекста

**Алгоритм**:
1. Строим n-gram модель из тренировочного набора
2. Храним частоты биграмм: `{(word1, word2): count}`
3. При коррекции учитываем предыдущее слово
4. Выбираем кандидата с максимальной биграммной частотой

**Пример**:
```python
# Биграммы из корпуса:
bigrams = {
    ("the", "cat"): 150,
    ("the", "car"): 80,
    ("a", "cat"): 200,
    # ...
}

# Коррекция "the cta":
# Кандидаты: ["cat", "car", "cut"]
# С учётом контекста "the":
#   - ("the", "cat"): 150 ← выбираем
#   - ("the", "car"): 80
#   - ("the", "cut"): 5
```

**Преимущества**:
- Учитывает контекст (улучшение точности на 2-3%)
- Решает проблему омофонов ("their" vs "there")
- Естественная интеграция с language model

**Недостатки**:
- Требует построения n-gram модели
- Больше памяти (~10-20MB для биграмм)
- Не работает для первого слова в предложении

**Оценка трудозатрат**: 2-3 дня

**Приоритет**: MEDIUM (улучшение точности)

**Альтернатива**: Интеграция KenLM (более мощная language model)

---

### 🔄 v4.0: Prefix Tree (Trie) (приоритет: LOW-MEDIUM)

**Цель**: Экономия памяти + O(m) поиск (m - длина слова)

**Подход**: Хранение словаря в префиксном дереве

**Структура**:
```python
class TrieNode:
    children: dict[str, TrieNode]  # {char: node}
    is_word: bool
    frequency: int

# Пример дерева для ["cat", "car", "dog"]:
#       root
#      /    \
#     c      d
#     |      |
#     a      o
#    / \     |
#   t   r    g
#  (✓) (✓)  (✓)
```

**Алгоритм поиска**:
1. Рекурсивный обход дерева с учётом edit distance
2. На каждом уровне проверяем: match, insert, delete, substitute
3. Собираем все слова в пределах max_distance

**Преимущества**:
- Экономия памяти (общие префиксы)
- O(m) поиск по префиксу
- Гибкость (можно добавлять слова динамически)
- Поддержка автодополнения

**Недостатки**:
- Сложная реализация (рекурсивный обход с edit distance)
- Медленнее Symspell для точного поиска
- Требует больше памяти на узлы дерева

**Компромисс с `words_by_len`**:
- **Вариант 1**: Одно дерево для всех слов (проще, но медленнее)
- **Вариант 2**: Дерево для каждой длины (быстрее, но больше памяти)
- **Рекомендация**: Вариант 1 (Trie уже даёт O(m), дополнительная оптимизация не нужна)

**Оценка трудозатрат**: 3-4 дня

**Приоритет**: LOW-MEDIUM (экономия памяти не критична для словаря 4000 слов)

**Рекомендация**: Отложить до увеличения словаря (>50k слов)

### 🔄 v5.0: Внешний словарь (приоритет: MEDIUM)

**Цель**: Расширение покрытия словаря

**Подход**: Интеграция внешнего английского словаря

**Источники**:
1. **SCOWL** (Spell Checker Oriented Word Lists)
   - URL: http://wordlist.aspell.net/
   - Размер: ~100k слов
   - Лицензия: Public Domain

2. **NLTK words corpus**
   - Размер: ~235k слов
   - Простая интеграция через Python

3. **Google 10000 English**
   - URL: https://github.com/first20hours/google-10000-english
   - Размер: 10k самых частых слов
   - Хорошо для начала

**Стратегия интеграции**:
```python
class EnhancedSpellChecker:
    def __init__(self, train_vocab, external_vocab):
        # Приоритет: train_vocab > external_vocab
        self.train_vocab = train_vocab      # 4k слов, высокий приоритет
        self.external_vocab = external_vocab # 10-100k слов, низкий приоритет
    
    def find_closest_word(self, word):
        # 1. Проверяем train_vocab (O(1))
        if word in self.train_vocab:
            return word
        
        # 2. Ищем коррекцию в train_vocab (приоритет)
        candidates_train = self._find_candidates(word, self.train_vocab)
        if candidates_train:
            return candidates_train[0]
        
        # 3. Проверяем external_vocab (O(1))
        if word in self.external_vocab:
            return word
        
        # 4. Ищем коррекцию в external_vocab
        candidates_ext = self._find_candidates(word, self.external_vocab)
        return candidates_ext[0] if candidates_ext else word
```

**Преимущества**:
- Покрытие редких слов (имена, термины)
- Улучшение recall (меньше пропущенных ошибок)
- Простая интеграция

**Недостатки**:
- Больше памяти (~1-10MB)
- Медленнее поиск (больше кандидатов)
- Риск ложных срабатываний (слова не из домена)

**Рекомендации**:
1. Начать с Google 10000 English (10k слов)
2. Фильтровать по частоте (только top 5k)
3. Использовать только для проверки наличия слова (не для коррекции)

**Оценка трудозатрат**: 1 день

**Приоритет**: MEDIUM (улучшение покрытия)

## Дополнительные улучшения

### 🔄 v6.0: Словарь частых ошибок OCR (приоритет: HIGH)

**Цель**: Быстрая коррекция типичных ошибок модели

**Подход**: Анализ ошибок модели + hardcoded словарь

**Алгоритм**:
1. Анализируем предсказания модели на validation set
2. Находим частые ошибки: `{predicted: ground_truth}`
3. Добавляем в spell checker как приоритетный словарь

**Пример**:
```python
common_ocr_mistakes = {
    # Путаница символов
    "0": "o",  # ноль → буква o
    "1": "l",  # единица → буква l
    "5": "s",  # пятёрка → буква s
    
    # Частые опечатки
    "teh": "the",
    "recieve": "receive",
    "occured": "occurred",
    
    # Специфичные для модели
    "beenn": "been",
    "peets": "peers",
    # ... (анализировать из логов)
}

class OCRAwareSpellChecker:
    def correct_word(self, word):
        # 1. Проверяем частые ошибки (O(1), приоритет)
        if word in self.common_mistakes:
            return self.common_mistakes[word]
        
        # 2. Обычная коррекция
        return super().correct_word(word)
```

**Преимущества**:
- Мгновенная коррекция (O(1))
- Специфично для вашей модели
- Улучшение WER на 1-2%

**Недостатки**:
- Требует анализа ошибок модели
- Нужно обновлять при переобучении

**Оценка трудозатрат**: 1 день

**Приоритет**: HIGH (быстрый эффект)

### 🔄 v7.0: Weighted Levenshtein (приоритет: MEDIUM)

**Цель**: Учёт типичных ошибок OCR в distance

**Подход**: Разные веса для разных типов ошибок

**Алгоритм**:
```python
def weighted_levenshtein(word1, word2, error_weights):
    """
    Levenshtein с учётом типичных ошибок OCR
    
    error_weights: {(char1, char2): weight}
    """
    dist = 0
    for c1, c2 in zip(word1, word2):
        if c1 != c2:
            # Если это типичная ошибка OCR - меньший вес
            weight = error_weights.get((c1, c2), 1.0)
            dist += weight
    return dist

# Пример весов:
error_weights = {
    ('o', '0'): 0.3,  # o → 0 (частая ошибка)
    ('l', '1'): 0.3,  # l → 1
    ('s', '5'): 0.4,  # s → 5
    # ... остальные веса = 1.0
}
```

**Преимущества**:
- Учитывает специфику OCR
- Улучшение точности на 1-2%

**Недостатки**:
- Требует анализа ошибок
- Сложнее в реализации

**Оценка трудозатрат**: 2 дня

**Приоритет**: MEDIUM (после анализа ошибок)
